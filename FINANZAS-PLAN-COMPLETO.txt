----------------------------------------------------------------------
1.1 SELECCION DE VISITAS PARA PAGAR
----------------------------------------------------------------------

PROBLEMA:
  Hoy el admin escribe un monto a mano y ya. No sabe que visitas pago.
  Puede pagar doble. No hay trazabilidad.

SOLUCION BACKEND:
  - Agregar campo payment_id (FK a staff_payment, nullable) en VisitHistory
  - Migracion: ALTER TABLE visit_history ADD COLUMN payment_id INTEGER
    REFERENCES staff_payment(id)
  - POST /staff-payments/ ahora recibe visit_ids: [1,2,3...]
  - Al crear pago: UPDATE visit_history SET payment_id = {new_id}
    WHERE id IN (visit_ids)
  - GET /staff-payments/summary calcula earned SOLO de visitas
    donde payment_id IS NULL (sin pagar)
  - GET /appointments/ agrega filtro: ?payment_status=unpaid|paid|all

SOLUCION FRONTEND:
  - En la lista expandida de visitas de cada staff:
    * Checkbox por cada visita
    * "Seleccionar todas sin pagar" / "Deseleccionar"
    * Contador: "12 visitas seleccionadas"
    * Monto auto-calculado: sum(comision + propina + productos)
  - Al abrir modal de pago:
    * Monto pre-llenado con el calculo
    * Lista resumida de visitas incluidas

  - Filtro en la lista: "Sin pagar" | "Pagadas" | "Todas"
