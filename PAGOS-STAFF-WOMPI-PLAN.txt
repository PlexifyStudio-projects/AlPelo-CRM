================================================================================
  3. INTEGRACION WOMPI — COMO FUNCIONA
================================================================================

Wompi (by Bancolombia) es el proveedor de pagos #1 en Colombia.
URL: https://wompi.com | Docs: https://docs.wompi.co

TIPOS DE OPERACION:
1. COBROS (recibir dinero de clientes):
   - Tarjeta de credito/debito
   - Nequi
   - PSE (transferencia bancaria)
   - Corresponsal bancario
   - Esto es para cuando implementemos cobros online (booking)

2. DISPERSIONES (enviar dinero a staff):
   - API de dispersiones: POST /v1/disbursements
   - Requiere cuenta bancaria destino verificada
   - Soporta: Bancolombia, Davivienda, Nequi, Daviplata, etc.
   - Respuesta inmediata o asincrona (webhook)

FLUJO DE DISPERSION:
1. Backend llama: POST https://api.wompi.co/v1/disbursements
2. Headers: Authorization: Bearer {private_key}
3. Body:
   {
     "amount_in_cents": 75200000,  // $752,000 COP (en centavos)
     "currency": "COP",
     "payment_method": {
       "type": "NEQUI",
       "phone_number": "3001234567"
     },
     // O para cuenta bancaria:
     "payment_method": {
       "type": "BANCOLOMBIA_TRANSFER",
       "bank_account": {
         "bank_name": "Bancolombia",
         "account_type": "SAVINGS",
         "account_number": "12345678901"
       }
     },
     "reference": "PAGO-STAFF-7-2026-04-02",
     "customer_data": {
       "full_name": "Angel Pabon",
       "phone_number": "3001234567",
       "legal_id": "1098765432",
       "legal_id_type": "CC"
     }
   }
4. Respuesta:
   {
     "data": {
       "id": "dis_abc123",
       "status": "PENDING",
       "amount_in_cents": 75200000,
       "payment_method": {...},
       "reference": "PAGO-STAFF-7-2026-04-02"
     }
   }
5. Webhook notifica cuando el pago es APPROVED o DECLINED

COSTOS WOMPI:
- Dispersiones: ~$800-1500 COP por transaccion
- Sin costo mensual fijo
- Requiere RUT/NIT del negocio + cuenta bancaria verificada

ALTERNATIVAS A WOMPI:
- Bold: Tambien colombiano, buena API
- Payvalida: Dispersiones masivas
- Kushki: Mas enterprise
- RECOMENDACION: Wompi por ser de Bancolombia y tener la API mas limpia

================================================================================
  5. FLUJO COMPLETO DE PAGO
================================================================================

CASO A: CON WOMPI INTEGRADO
1. Admin abre Nomina > ve saldo de Angel: $752,000
2. Click "Pagar" > se abre modal
3. Modal muestra:
   - Datos de Angel (nombre, cedula, cuenta)
   - Monto a pagar (editable)
   - Metodo: Nequi de Angel (3001234567) [seleccion automatica]
   - Concepto auto-generado
4. Admin confirma > click "Pagar $752,000"
5. Backend:
   a. Crea StaffPayment con status="processing"
   b. Llama API Wompi POST /disbursements
   c. Wompi responde con ID de transaccion
   d. Se actualiza StaffPayment con referencia Wompi
6. Webhook de Wompi notifica:
   a. Si APPROVED: status="paid", se notifica por WhatsApp al staff
   b. Si DECLINED: status="failed", se notifica al admin
7. Se genera comprobante de pago descargable

CASO B: SIN WOMPI (PAGO MANUAL)
1. Admin abre Nomina > ve saldo de Angel: $752,000
2. Click "Pagar" > se abre modal
3. Modal muestra:
   - Datos bancarios de Angel para que el admin haga la transferencia
   - "Transferir a: Nequi 300-123-4567 (Angel Pabon)"
4. Admin hace la transferencia desde su app bancaria
5. Vuelve al modal:
   - Pega la referencia de la transferencia
   - Sube foto del comprobante (opcional)
   - Click "Confirmar pago"
6. Se registra StaffPayment + se notifica por WhatsApp

CASO C: PAGO EN EFECTIVO
1. Admin paga en efectivo al staff en persona
2. Abre Nomina > click "Pagar" > selecciona "Efectivo"
3. Registra el pago como constancia
4. Se notifica por WhatsApp como confirmacion


================================================================================
  6. SEGURIDAD
================================================================================

- Datos bancarios encriptados en BD (AES-256 o similar)
- API keys de Wompi encriptadas (igual que AI provider keys)
- Solo admin puede ver/editar datos bancarios
- Staff NO puede ver su propia cuenta desde el portal staff
- Cada transaccion tiene referencia unica
- Logs de auditoria por cada pago
- Doble confirmacion antes de dispersar (modal de confirmacion)
- Rate limiting en dispersiones (max 10 por hora)
- Webhook de Wompi verificado con events_key (firma HMAC)
- Tenant isolation: cada negocio solo ve sus pagos

  7. CONFIGURACION POR TENANT
================================================================================

En Settings > seccion "Pagos y Dispersiones":

CAMPO                    | TIPO        | DESCRIPCION
-------------------------|-------------|------------------------------------
payments_provider        | select      | Wompi / Manual / Desactivado
wompi_public_key         | password    | pub_test_xxx o pub_prod_xxx
wompi_private_key        | password    | prv_test_xxx o prv_prod_xxx
wompi_events_key         | password    | Para verificar webhooks
wompi_environment        | select      | Sandbox (pruebas) / Produccion
auto_notify_staff        | boolean     | Enviar WhatsApp automatico al pagar
payment_approval_required| boolean     | Requiere aprobacion de otro admin
default_payment_method   | select      | Nequi / Bancolombia / Efectivo

FLUJO DE ACTIVACION:
1. Negocio se registra en wompi.com con su RUT
2. Wompi verifica y activa la cuenta (1-3 dias)
3. Admin copia las API keys a Settings > Pagos
4. Selecciona "Produccion" como ambiente
5. Listo — ya puede dispersar desde la plataforma
